# Observation Schemas

---
类所在位置：

```
oxygent/schemas/observation.py
```

---

## 简介

本模块定义了工具执行观测结果的数据结构。`Observation` 收集一个或多个 `ExecResult` 条目——每个条目代表单次工具调用的输出——并提供格式化显示功能。

## ExecResult (BaseModel)

| 参数           | 类型          | 默认值     | 描述                              |
| -------------- | ------------- | ---------- | --------------------------------- |
| `executor`     | `str`         | （必填）   | 工具/执行器的名称。              |
| `oxy_response` | `OxyResponse` | （必填）   | 工具返回的响应。                 |

## Observation (BaseModel)

### 参数

| 参数           | 类型               | 默认值  | 描述                                 |
| -------------- | ------------------ | ------- | ------------------------------------ |
| `exec_results` | `List[ExecResult]` | `[]`    | 各工具执行结果的列表。               |

### 方法

| 方法                                | 协程 (async) | 返回值       | 用途（简述）                                                           |
| ----------------------------------- | ------------ | ------------ | ---------------------------------------------------------------------- |
| `add_exec_result(exec_result)`      | 否           | `None`       | 追加一个执行结果。                                                     |
| `to_str(is_prefix_included=True)`   | 否           | `str`        | 将所有结果格式化为可读字符串，可选择是否包含工具名称前缀。             |
